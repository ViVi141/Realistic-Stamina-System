## 机制概览

玩家穿过灌木时变慢，**不是**脚本每帧做距离检测，而是：

1. **引擎**通过角色的 `Foliage` 碰撞层，检测与植被实体的接触；
2. 接触列表变化时，调用脚本回调 `OnSpecialContactsChagned`；
3. **游戏脚本**（`SCR_ChimeraCharacter`）把接触的 `Tree` 实体换算成「有效植被高度」；
4. 用**曲线**把高度映射为速度比例（0~1）；
5. 通过 `CharacterControllerComponent.OverrideMaxSpeed()` 限制角色最大速度；
6. 减速过程经 `SCR_CharacterSlowdownEasingSystem` 做平滑插值。

同一套「特殊接触」管线还处理铁丝网、地雷等带 `SCR_SpecialCollisionHandlerComponent` 的物体，但**灌木减速**走的是 `Tree` 分支，不依赖该组件。

## 运行时流程

### 阶段 0：初始化

角色生成时加载 `SCR_ChimeraCharacterClass` 上的配置：

| 配置项 | 位置 | 作用 |
|--------|------|------|
| `m_sBushSlowTypeCfgPath` | 角色 Class | 指向 `Configs/Character/BushTypeSlow.conf` |
| `m_cFoliageHeightSlowCurve` | 角色 Class / `Character_Base.et` | 高度 → 速度比例曲线 |
| `FoliageCollisionLayer "Foliage"` | `Character_Base.et` → `CharacterSoundComponent` | 声明参与植被碰撞检测 |

`SCR_ChimeraCharacter` 实例还维护：

- `m_mSpeedReferences`：多个减速来源（灌木、铁丝网等）及各自限制值；
- `m_fTargetSpeed` / `m_fCurrentSpeed`：目标与实际生效的速度比例（默认 1 = 全速）；
- `m_aContacts`：当前接触的特殊碰撞实体列表（含铁丝网等）。

---

### 阶段 1：引擎检测接触

角色移动时，引擎根据 **`FoliageCollisionLayer`** 与场景中植被的碰撞体，维护当前接触的实体集合。

当集合发生变化时，对角色调用（由角色本地拥有者侧驱动）：

```c
// scripts/Game/generated/Character/ChimeraCharacter.c
event void OnSpecialContactsChagned(notnull array<IEntity> contacts);
```

`SCR_ChimeraCharacter` 重写该事件：

```c
// scripts/GameCode/Character/SCR_ChimeraCharacter.c
override void OnSpecialContactsChagned(notnull array<IEntity> contacts)
```

> **注意**：方法名在生成代码中为 `OnSpecialContactsChagned`（拼写 Chagned），与文档/注释一致即可。

---

### 阶段 2：同步离开的旧接触

若 `m_aContacts` 中某实体已不在新的 `contacts` 列表里：

1. 非 Authority 端通过 `Rpc_ContactChanged` 通知服务器移除；
2. 调用 `RemoveOldContact`（处理伤害效果、音频信号等）；
3. 从 `m_aContacts` 移除。

---

### 阶段 3：处理新接触（两条分支）

对每个 `contacts` 里、且尚未记录在 `m_aContacts` 的实体：

#### 分支 A：特殊碰撞物（铁丝网、地雷等）

若实体带有 `SCR_SpecialCollisionHandlerComponent`：

- 加入 `m_aContacts`；
- Authority 上可能挂载 `SCR_SpecialCollisionDamageEffect`；
- 设置音频信号 `SpecialContact`、`SpecialContactEntityHeight`；
- **不**在本函数内直接算灌木减速（铁丝网等另有 `SetSpeedLimit` 来源）。

#### 分支 B：植被（灌木 / 树）——减速主逻辑

```c
currentTree = Tree.Cast(newContact);
if (!currentTree)
    continue;  // 不是 Tree，跳过

newContact.GetBounds(mins, maxs);
currentTreeType = TreeClass.Cast(currentTree.GetPrefabData()).SoundType;
evaluationModHeight = GetModifiedBushHeight(maxs[1] - mins[1], currentTreeType);

if (evaluationModHeight > highestModifiedHeight)
    highestModifiedHeight = evaluationModHeight;
```

要点：

- 灌木预制体继承 `Tree`（见 `Bush_Base.et`），因此用 `Tree.Cast` 识别；
- **包围盒高度** `maxs[1] - mins[1]` 表示植被实体在世界中的竖直尺寸；
- `SoundType`（`ETreeSoundTypes`）决定该品种的「茂密程度」权重；
- 同时接触多丛灌木时，取 **最大的** `evaluationModHeight`（最碍事的一丛主导减速）。

---

### 阶段 4：计算并设置减速

```c
CalculateAndSetSlowdown(highestModifiedHeight);
```

#### 4.1 类型加权高度

```c
protected float GetModifiedBushHeight(float foliageHeight, ETreeSoundTypes treeType)
{
    SCR_ChimeraCharacterClass classData = SCR_ChimeraCharacterClass.Cast(GetPrefabData());
    return foliageHeight * classData.GetBushSlowTypeCfg().GetHeightSlowEffectModifier(treeType);
}
```

`GetHeightSlowEffectModifier` 查表 `Configs/Character/BushTypeSlow.conf`：

| SoundType | m_fHeightModifier（示例） | 含义 |
|-----------|---------------------------|------|
| Bush_Small | 1.0（默认） | 基准 |
| Bush（默认条目） | 1.3 | 更密 → 等效更高 → 更慢 |
| Bush_Leafy | 1.25 | 阔叶灌木 |
| Bush_Reed | 0.9 | 芦苇类较疏 |
| Conifer | 1.0 | 针叶灌木 |
| Leafy / LeafyDomestic | 0.5 | 乔木冠层较疏，少拖慢 |

未在表中列出的类型使用 `m_fDefaultSlowHeightModifier`（默认 1）。

#### 4.2 曲线映射为速度比例

```c
float bushSlow = LegacyCurve.Curve(
    ECurveType.CurveProperty2D,
    bushModdedHeight,
    classData.GetFoliageHeightSlowdownCurve())[1];

SetSpeedLimit(this, bushSlow);
```

`Character_Base.et` 中的曲线示例：

```
m_cFoliageHeightSlowCurve {
  0 1 0.04 1 0.18 0.7 0.4 0.5 0.8 0.45 1 0.599
}
```

可读作（近似）：

| 修正高度 | 速度比例 | 效果 |
|----------|----------|------|
| 0 | 1.0 | 几乎不减速 |
| 很低 | ~1.0 | 矮灌丛影响小 |
| 中等 | ~0.5~0.7 | 明显减速 |
| 很高 | ~0.45~0.6 | 大树冠；曲线末端略回升，减轻乔木对移动的拖累 |

曲线 Y 轴为 **最大速度的比例**（1 = 100% 原速，0.5 ≈ 半速）。

---

### 阶段 5：多来源合并 + 应用速度

#### 5.1 SetSpeedLimit

```c
void SetSpeedLimit(Managed source, float limit)
```

- 以 `source`（如 `this` 代表灌木）为键，存入 `m_mSpeedReferences`；
- 遍历所有来源，取 **最小** `limit`（最严格的限制生效）；
- `limit == 1` 的项会被移除（表示该来源不再限速）；
- 结果写入 `m_fTargetSpeed`。

灌木、铁丝网等可同时存在；玩家始终按 **最慢** 的那条规则移动。

#### 5.2 减速 vs 加速

```c
if (m_fTargetSpeed < m_fCurrentSpeed)   // 需要变慢
    characterSlowdownSystem.Register(this);  // 交给缓动系统
else                                    // 需要变快（离开灌木）
    characterSlowdownSystem.Unregister(this);
    OverrideSpeed(speedLimit);               // 立即恢复
```

- **进入灌木**：逐渐 Lerp 到目标速度；
- **离开灌木**：**立即** 提高到目标速度（手感更干脆）。

#### 5.3 OverrideSpeed → 引擎

```c
protected void OverrideSpeed(float speed)
{
    m_fCurrentSpeed = speed;
    GetCharacterController().OverrideMaxSpeed(speed);
}
```

`OverrideMaxSpeed` 说明（生成 API）：

> 将角色最大速度覆盖为 **当前最大速度的一个比例**。  
> `value < 0` 不覆盖；`value > 1` 使用正常最大速度。

即：`0.5` 表示最多跑到设定最大速度的 50%。

---

### 阶段 6：平滑过渡（仅减速时）

`SCR_CharacterSlowdownEasingSystem` 在 `WorldSystemPoint.FixedFrame` 更新：

```c
void UpdateSlowdown(float transitionTime)
{
    float newSpeed = Math.Lerp(m_fCurrentSpeed, m_fTargetSpeed, transitionTime);
    // transitionTime = dt * m_fTransitionStrength（默认 Strength = 3）
    OverrideSpeed(newSpeed);
}
```

当 `newSpeed` 与 `m_fTargetSpeed` 相差 < 0.01 时，取消注册，停止更新。

```
        │
        ▼

        │
        ▼
OnSpecialContactsChagned(contacts)
        │
        ├─► 移除已离开接触 → RemoveOldContact

                │
                └─ 是 Tree?

                        ┌───────────────────────┴───────────────────────┐

              m_fTargetSpeed < m_fCurrentSpeed              m_fTargetSpeed >= m_fCurrentSpeed
                        │                                               │
                        ▼                                               ▼
        SCR_CharacterSlowdownEasingSystem.Register          立即 OverrideMaxSpeed
                        │
                        ▼
              每帧 Lerp → OverrideMaxSpeed
                        │
                        ▼
              达到目标 → Unregister
```

## 减速强度如何计算

公式（概念上）：

```
修正高度 = 包围盒高度 × GetHeightSlowEffectModifier(SoundType)
速度比例 = Curve(修正高度)   // m_cFoliageHeightSlowCurve 的 Y 值
实际最大速度 = 原最大速度 × min(所有 SetSpeedLimit 来源的速度比例)
```

**不直接使用** `m_iFoliageHeight` 做移动减速；该字段在 `Tree.c` 中标注为 foliage 离地距离，主要用于环境音、昆虫等。移动减速用的是 **实体整体包围盒高度** + **SoundType 权重**。

---

## 速度限制如何应用

- 灌木作为 `SetSpeedLimit(this, bushSlow)` 的 `source`，离开灌木后若 `bushSlow` 回到 1，该来源从 map 移除；
- 与铁丝网等其他 `source` 独立，取全局最小比例；
- 最终唯一出口：`GetCharacterController().OverrideMaxSpeed(fraction)`。

## 平滑过渡系统

| 属性 | 默认值 | 文件 |
|------|--------|------|
| `m_fTransitionStrength` | 3 | `SCR_CharacterSlowdownEasingSystem.c` |

越大，每帧 Lerp 步长越大，减速体感越快。仅在 **目标速度低于当前速度** 时启用；加速离开无缓动。

## 灌木预制体与植被数据

`Bush_Base.et` 结构要点：

```et
Tree {
  components {
    RigidBody {
      ModelGeometry 1   // 用模型网格做碰撞
      Static 1
    }
  }
  SoundType Bush        // 默认音效/减速类型
}
```

具体品种（如 `b_juniperus_communis_2s.et`）可覆盖 `SoundType`（如 `Conifer`）、`MaxHealth`、碰撞倍率等。  
**碰撞几何**决定玩家何时算「在灌木里」；**SoundType** 决定减速算多重。

## 与音效系统的关系

移动减速与灌木摩擦音 **共用引擎 Foliage 接触检测**，但脚本路径不同：

| 系统 | 入口 | 输出 |
|------|------|------|
| 移动减速 | `SCR_ChimeraCharacter.OnSpecialContactsChagned` | `OverrideMaxSpeed` |
| 音效 | `CharacterSoundComponent` + `Character_Collisions.sig` | `BushLayer_V`、`IsInBush`、`BushContact` 等信号 |

`Sounds/Character/Character_Collisions.sig` 中有 `BushContact`、`BushHeight`、`BushLayer_V` 等，驱动 `Character_Movement_Foley_*.acp` 的沙沙声，**不**参与速度计算。

---

## 联机与复制

- `OnSpecialContactsChagned` 注释写明：**由角色拥有者本地调用**；
- 接触变化时，非 Authority 通过 `Rpc_ContactChanged` 同步 `m_aContacts` 的增删；
- `SetSpeedLimit` / `OverrideMaxSpeed` 在本地控制器上执行，实际移动仍受引擎移动与复制约束。

修改减速逻辑时需考虑：仅改客户端或仅改服务器可能导致不同步，应在与现有 `Rpc_ContactChanged` 相同权威模型下测试。

## 如何修改手感

### 1. 调整整体「高度 → 速度」曲线

编辑 `Prefabs/Characters/Core/Character_Base.et`（或派生角色预制体）：

```
m_cFoliageHeightSlowCurve { ... }
```

降低曲线中段 Y 值 → 同样灌木更慢；提高 → 更易穿行。

### 2. 调整某类灌木的茂密权重

编辑 `Configs/Character/BushTypeSlow.conf`，为对应 `ETreeSoundTypes` 增加或修改 `SCR_BushContainer` 条目及 `m_fHeightModifier`。

同时确保该灌木 `.et` 的 `SoundType` 与配置一致。

### 3. 调整减速/恢复过渡

- 减速快慢：`SCR_CharacterSlowdownEasingSystem` 的 `m_fTransitionStrength`；
- 离开恢复：当前为即时恢复，若要渐变需改 `SetSpeedLimit` 中 `else` 分支逻辑。

### 4. 调整碰撞范围（何时算进入灌木）

改灌木模型的碰撞网格 / `RigidBody` `ModelGeometry`，或 `"Collision multiplier"` 等，影响 **何时触发** 接触，而非曲线形状。

## 常见问题

**Q：为什么搜不到 `IsInBush` 的脚本？**  
A：`IsInBush` 出现在 `Character_Collisions.sig` 音频信号图里，由引擎/信号驱动音效，不是 Enforce 移动逻辑里的变量名。

**Q：矮灌木几乎不减速？**  
A: 曲线在低高度时 Y≈1；且包围盒高度小，修正高度低，映射结果接近全速。

**Q：树和灌木都继承 Tree，大树也会减速吗？**  
A: 会。高大 `Tree` 包围盒高，但 `Leafy` 等类型权重可设为 0.5，且曲线高端 Y 会回升，减轻乔木影响。

**Q：能否在 Mod 里完全禁用灌木减速？**  
A: 可将曲线设为全 `1`，或重写 `SCR_ChimeraCharacter.OnSpecialContactsChagned` 中 Tree 分支；也可将 `m_sBushSlowTypeCfgPath` 指向全 0 权重的配置（需自行保证曲线 Y 为 1）。

**Q：`m_iFoliageHeight` 改大能否让玩家更慢？**  
A: 对 **移动减速** 几乎无直接影响；该值主要用于音效等。移动看包围盒高度与 `SoundType`。

## 13. RSS 模组兼容（Realistic Stamina System）

RSS 每帧根据体力计算速度倍率。若直接调用 `CharacterControllerComponent.OverrideMaxSpeed()`，会**覆盖**上一帧原生 `SCR_ChimeraCharacter` 经 `SetSpeedLimit` / 缓动系统写入的灌木限速。

### 正确做法

RSS 通过 `SCR_RSS_CharacterSpeedBridge.ApplyStaminaSpeedLimit()` 写入**独立 source**（`RSS_StaminaSpeedLimitToken`），参与与灌木（`SetSpeedLimit(this, bushSlow)`）、铁丝网（`SCR_SpecialCollisionDamageEffect`）等的 **min 合并**：

```
实际 OverrideMaxSpeed = min(RSS 体力倍率, 灌木曲线倍率, 铁丝网倍率, …)
```

实现文件：

| 文件 | 作用 |
|------|------|
| `scripts/Game/RSS/Core/SCR_RSS_CharacterSpeedBridge.c` | 桥接：RSS → `SetSpeedLimit` |
| `scripts/Game/Integration/PlayerBase.c` | 玩家/AI 主循环速度应用 |
| `scripts/Game/RSS/Core/SCR_StaminaUpdateCoordinator.c` | 速度计算后应用 |
| `scripts/Game/RSS/AI/SCR_RSS_AISpeedCap.c` | AI 体力状态限速 |

### 依赖

需要新版游戏脚本中的 `SCR_ChimeraCharacter.SetSpeedLimit()` 与 `SCR_CharacterSlowdownEasingSystem`（见 `scripts/GameCode/Character/SCR_ChimeraCharacter.c`）。旧版无 `SetSpeedLimit` 时桥接回退为直接 `OverrideMaxSpeed`（无灌木合并）。
