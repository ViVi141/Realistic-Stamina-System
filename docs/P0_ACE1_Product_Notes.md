# P0-ACE-1：数据导出与「速度对照」产品立场

## 1. 当前设计（代码与契约）

- **RPC** `RPC_ClientReportStamina`（见 `scripts/Game/Integration/PlayerBase.c`）带有 **`//!`** 说明：在 **`m_bDataExportEnabled`** 打开时，用于**客户端向服务端上报**体力/重量等，供服务端与 JSON 数据导出等**对照/诊断**；**不是**「强反作弊」或**玩法内唯一权威**判据。
- **关导出**：服务端在 RPC 内直接 `return`；客户端侧是否发送由现有逻辑与 `SetServerDataExportEnabled` 等门闸控制（以当前实现为准）。

## 2. 产品决策（与竞技/公开服）

| 策略 | 说明 |
| --- | --- |
| **推荐默认心智** | 将「数据导出 + 体力上报」定位为 **私服/分析/对比工具链**，不对外承诺「防作弊或竞技公平性」。 |
| **公开/竞技服** | 若仍开启 `m_bDataExportEnabled`，应视为**可观测性/运维权衡**；对作弊面的要求需单独做服务端权威重算、限制或关断（超出本清单范围时可另开任务）。 |
| **默认 JSON** | 是否默认 `m_bDataExportEnabled` 为真/假由 **`RealisticStaminaSystem.json` 与你们发行策略** 决定；本说明不强制改默认值。 |

## 3. 与审计待办 P0-ACE-1 的对应

- **「明确定义威胁模型」**：本文档为**产品/运维层**的固定结论；若需**工程级**加强（白名单、仅本地信任、服务器唯一步进等），应另列设计与验收。
- **验收**：关闭导出时无 RPC 负载或可忽略；打开时行为符合上节「诊断」定位——由 P0-ACE-1 原文中的安全测试补充。
