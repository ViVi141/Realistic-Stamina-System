Realistic Stamina System (RSS) is a realistic stamina and locomotion mod for **Arma Reforger**.
The current version uses a **Pandolf / ACSM / Critical PowerвЂ“WвЂІ** stack to drive speed limiting, drain, recovery, and sprint eligibility.

## Overview

The v6 player path is built around:

```text
v_meas -> metabolic power -> effective CP / W' -> speed limit -> SetSpeedLimit
```

That means:

- drain accounting uses **measured speed**
- sprint is **not** fixed `Run Г— 1.30`
- the player path has **no willpower plateau**
- speed limits go through `SCR_RSS_SpeedBridge` so foliage and wire slowdown still merge by `min`

## Current behavior

### Movement

- Player speed is driven by `SCR_RSS_SpeedCalculator.CalculateV6PhaseSpeedMultiplier()`; above the limp threshold, stamina itself does not apply the old plateau/decay curve, but Run can still be pushed down by CP/WвЂІ metabolic limits.
- Limp starts below `SMOOTH_TRANSITION_END = 0.05`.
- `SCR_RSS_CollapseTransition` adds a 5-second damping phase after crossing the limp threshold.
- Current runtime gait tops fall back to engine-aligned values because `V6_USE_MARCH_GAIT_SPEEDS = false`:
  - Walk: about `1.45 m/s`
  - Run: about `3.8 m/s`
  - Sprint: about `5.5 m/s`

### Sprint and WвЂІ

- Sprint depends on gait target, available power, WвЂІ pool, and the minimum sprint-over-run separation.
- Aerobic sprint threshold fallback: `0.25`
- WвЂІ sprint threshold fallback: `0.20`
- `SPRINT_GAIT_MIN_OVER_RUN_RATIO = 1.25`
- Sprint drain is not a simple fixed `3x` multiplier in the current model.

### Encumbrance and slope

- Encumbrance affects both speed and the metabolic / CP budget.
- Fallback speed penalty constants:
  - coefficient: `0.28`
  - exponent: `1.5`
- Runtime values normally come from presets.
- Slope pacing uses a Tobler-style adjustment, not a flat вЂњpercent per degreeвЂќ rule.

### Recovery and fatigue

- Recovery is computed as a net metabolic result.
- Moving characters normally do not receive aerobic recovery, except for the low-stamina walk-recovery zone used on land below about `15%` STA when not sprinting.
- EPOC delay lasts `2 seconds` after stopping.
- Low-stamina rest gate fallback: `5 seconds`; the shipped preset defaults are typically `3 seconds`.
- Key fallbacks:
  - `BASE_RECOVERY_RATE = 0.00010`
  - `LOAD_RECOVERY_PENALTY_COEFF = 0.0002`
  - `FATIGUE_ACCUMULATION_COEFF = 0.025`
  - `FATIGUE_MAX_FACTOR = 2.5`

### Environment and swimming

The temperature cache is generated by `SCR_RSS_AstronomyMath.CalculateUniversalTemperature()` and refreshed by `SCR_RSS_TemperatureSampler` every few seconds or after large position changes.

At a high level, the model is:

```text
T = latitude baseline + seasonal term + daily term - altitude lapse - rain cooling
```

Key details:

- latitude baseline: `27 - 42 * sin(lat)^2`
- seasonal term: cosine by day-of-year, with hemisphere sign
- daily term: sine by hour-of-day, with cloud / fog damping on daily range
- altitude lapse: about `6.5В°C / 1000m`
- rain cooling: additional cooling in warm conditions, capped at `5В°C`

That cached temperature then feeds:

- heat-stress multiplier above `26В°C`
- heat recovery penalty above `30В°C`
- energy compensation using effective temperature `T_eff = T - 1.35 * sqrt(windSpeed)`
- heat, cold, wind, rain wet weight, surface wetness, mud, and indoor detection all feed into stamina behavior
- Rain wet weight accumulates continuously from rain intensity.
- Total wet weight is `swim wet weight + rain wet weight`, clamped to `10 kg`.
- Swimming uses a separate model with:
  - drag-dominated dynamic cost
  - `SWIMMING_BASE_POWER = 20W` fallback
  - `SWIMMING_ENCUMBRANCE_THRESHOLD = 25kg`
  - up to `10kg` swim wet weight with `30s` decay after leaving water

### Jump / vault

- Jump cost is computed physically with `ComputeJumpCostPhys()`.
- Climb / vault cost is computed with `ComputeClimbCostPhys()`.
- Consecutive jump window: `2s`
- Jump cooldown: `2s`
- Jump is blocked below `10%` stamina.

### AI

The current AI path is:

```text
SCR_RSS_AIManager -> AIStaminaState -> AISpeedCap -> AIIntentFilter -> AICombatDecay
```

- Player update interval: `17ms`
- AI base update interval: `100ms`
- AI distance LOD: `200 / 300 / 1500ms`
- AI behavior layer throttle: `500ms`

## Installation

1. Put the project under the Workbench `addons` directory.
2. Open `addon.gproj`.
3. Build the project.
4. Enable the mod in-game or on the server.

## API example

```c
RSS_PlayerInfo info = SCR_RSS_API.GetPlayerInfo(player);
if (info.isValid)
{
    PrintFormat("STA=%1 W'=%2 sprintAllowed=%3",
        info.staminaPercent,
        info.wPrimePool01,
        info.sprintAllowed);
}
```

Use `wPrimePool01` instead of deprecated `anaerobicPercent`.
